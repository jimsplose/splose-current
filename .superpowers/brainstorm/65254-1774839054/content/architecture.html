Import Rules (Enforced by ESLint)

Rule Description Example
✅ Allowed Import from @/components/ds import { Button, Card } from '@/components/ds'
❌ Banned Direct import from antd in page/feature code import { Button } from 'antd'
❌ Banned Direct import from @ant-design/icons in page code import { EditOutlined } from '@ant-design/icons'
✅ Allowed Direct antd imports inside ds/ components only // inside ds/Button.tsx
import { Button as AntButton } from 'antd'
⚠️ Exception AntD layout components (Flex, Row, Col, Space) allowed in pages import { Flex, Row, Col } from 'antd'

The layout exception exists because wrapping Flex/Row/Col adds no value — they have no Splose-specific defaults. This keeps the DS focused on components with actual opinions.

Data Flow: Theme → Components → Pages

1. theme.ts — defines Splose token overrides (colors, fonts, sizes, radii)
↓
2. ThemeProvider.tsx — wraps app in <ConfigProvider theme={sploseTheme}>
↓
3. layout.tsx — renders <ThemeProvider>{children}</ThemeProvider>
↓
4. DS wrappers (Button.tsx, etc.) — import from antd, apply Splose-specific prop defaults
↓
5. Pages — import from @/components/ds, use components with Splose API
