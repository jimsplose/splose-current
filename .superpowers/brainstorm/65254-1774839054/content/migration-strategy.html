Section 4: Migration Strategy

How the migration rolls out across 43 components and 96 pages.

Migration Phases

1

Foundation

Set up the infrastructure before touching any components.

  • Install antd, @ant-design/icons, @ant-design/nextjs-registry
  • Create theme.ts with all Splose token overrides
  • Create ThemeProvider.tsx wrapping ConfigProvider
  • Wire ThemeProvider into layout.tsx
  • Add ESLint no-restricted-imports rule
  • Verify AntD renders correctly alongside existing Tailwind (temporary coexistence)

Tailwind still active. Both systems coexist briefly during this phase.

2

DS Component Rewrite

Rewrite all 43 components in ds/ to use AntD internally. Keep the same export names and prop APIs where possible.

  • Same-API wrappers first — Button, Card, Modal, Badge, Alert, Checkbox, Toggle, Spinner, Avatar (props barely change)
  • Adapted wrappers next — FormInput, FormSelect, FormTextarea, Select, Dropdown, Tab, Collapse (prop shape changes)
  • DataTable last — heaviest migration. Rewrite from primitive JSX table to AntD Table with column definitions. This changes how every page defines its table.
  • Custom components — rewrite to use AntD tokens + CSS Modules instead of Tailwind classes. No AntD component dependency, just visual consistency.
  • Rebuild all 43 Storybook stories against new implementations

Barrel export stays stable. Pages still import from @/components/ds — internal rewrites are invisible to consumers where APIs don't change.

3

Page Migration

Update all 96 pages to use the new DS APIs and remove Tailwind classes.

  • Simple pages first — settings pages, static layouts (mostly swapping Tailwind layout → AntD Flex/Row/Col)
  • Table pages next — rewrite inline .map() loops to AntD Table column definitions
  • Complex pages last — Dashboard, Calendar, DataImport wizard (most Tailwind-heavy)
  • Replace all Lucide icons with @ant-design/icons equivalents
  • Replace all Tailwind layout classes with AntD layout components

Page-by-page. Each page is independently migratable. No big-bang switchover.

4

Cleanup

Remove all traces of the old system.

  • Uninstall tailwindcss, @tailwindcss/postcss, prettier-plugin-tailwindcss, lucide-react
  • Delete postcss.config.mjs
  • Strip globals.css down to @font-face declarations only
  • Delete portable/ directory
  • Delete usePagination.ts
  • Verify build passes with zero Tailwind references
  • Final Storybook verification

Zero Tailwind left. Build should fail if any Tailwind class or import remains.

API Changes: What Breaks for Consumers

Most wrappers preserve the current API. These are the exceptions where page code must change.

Component Current API New API
DataTable Inline JSX: <Tr><Td>...</Td></Tr> with .map() Column definitions: columns={[...]} dataSource={data}
Th (sort) Props: sortable, sortDirection, onSort Column def: sorter: true, sortOrder
ExpandableRow Component: <ExpandableRow expandContent={...}> Table prop: expandable={{ expandedRowRender }}
FormInput Props: label, error on the input May wrap in Form.Item internally (API stays similar)
Dropdown Props: items: DropdownItem[] Maps to AntD menu items format internally (API stays similar)
Button, Card, Modal, Badge, etc. Current props Same props — wrapper maps internally to AntD

Red = page code must change. Yellow = minor adaptation. Green = no change needed.

Risk: Next.js App Router + AntD CSS-in-JS

Ant Design 5.x uses CSS-in-JS which requires client-side style injection. In Next.js App Router with server components, styles can flash (FOUC) on first render. The fix:

  • @ant-design/nextjs-registry extracts styles at build time for SSR
  • Wrap the app in <AntdRegistry> in layout.tsx alongside ThemeProvider
  • This is Ant Design's official solution — well-documented and actively maintained
