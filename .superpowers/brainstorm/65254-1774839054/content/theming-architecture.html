Section 2: Theming & Styling Architecture

How to make Ant Design 5.x look exactly like current Splose, with zero Tailwind.

Current State → Target State

Current: Tailwind CSS 4
Theme: CSS variables in globals.css
Styling: Tailwind utility classes on every element
Typography: Custom CSS classes (text-display-lg, text-body-md)
Colors: --color-primary: #8250ff, etc.
Fonts: Sprig Sans (display) + Inter (body)
Spacing: Tailwind spacing scale (p-4, gap-6)
Icons: Lucide React
Problem: Engineers copy Tailwind classes between files. No enforced consistency.
Target: Ant Design 5.x Token System
Theme: ConfigProvider with custom theme tokens
Styling: AntD's CSS-in-JS (cssinjs) — no utility classes
Typography: AntD Typography + custom Text wrapper
Colors: colorPrimary: '#8250ff' in theme config
Fonts: fontFamily: 'Inter' + custom Sprig Sans for display
Spacing: AntD Flex/Space components + theme tokens
Icons: @ant-design/icons
Result: Engineers use components with props. No raw styling decisions.

Sprig Sans: The Exception

Ant Design's token system handles one font family globally. Your display headings use Sprig Sans, which needs special treatment:

  • The Text wrapper component stays custom — it applies Sprig Sans for display variants and Inter for everything else
  • AntD's fontFamily token is set to Inter (the 95% case)
  • The Text component uses a CSS Module or createStyles to apply Sprig Sans only on display/lg, display/md, display/sm variants
  • This is the one place where a font-family override exists outside AntD's token system
