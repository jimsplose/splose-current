DataTable: Comprehensive Replacement Options

Your current DataTable is used across 25+ pages with 14 distinct patterns: sorting, filtering, pagination, expansion, row selection, actions dropdowns, link cells, responsive column hiding, and search integration. No inline editing today, but production Splose likely needs it.

Current State

Your DataTable is a primitive JSX table — no column definition system, no built-in sort/filter/pagination logic. All state management lives in each page component. Engineers manually wire .map() loops, useState for sort/expand/select, and the usePagination hook. This is the core of the copy-paste problem.

A

Ant Design Table (built-in) — Recommended

Zero extra dependencies. Ships with antd. Perfect theme integration.

Pros

  • No extra bundle cost — it's already in antd
  • Column definition system (no more .map() loops)
  • Built-in sort, filter, pagination, expand, selection
  • Perfect AntD theming — inherits ConfigProvider
  • Server-side pagination via onChange callback
  • Virtual scrolling (v5.4+) for large datasets
  • Excellent TypeScript, active maintenance (94k stars)

Cons

  • Inline editing requires custom wiring (Form + Table)
  • Column resizing needs react-resizable addon
  • Virtual scroll can't combine with expandable rows
  • No built-in CSV/Excel export

Covers ~80% of your 14 patterns out of the box. For the remaining 20% (inline editing, export), build targeted wrappers.

B

TanStack Table v8 (headless) + AntD rendering

Maximum flexibility. Logic engine only — you render with AntD components. ~17KB bundle.

Pros

  • Every feature works together (virtual + expand + edit)
  • You render AntD markup, so theme integration is perfect
  • Tiny bundle (~17KB core + ~5KB virtual)
  • Column definition system with full TypeScript generics
  • Global search, multi-sort, column visibility all built-in
  • No feature limitations or incompatibilities
  • MIT license, 26k stars, very popular

Cons

  • Headless = you build ALL the UI (sort arrows, filter dropdowns, pagination)
  • 3-5 days upfront to build a polished DataTable wrapper
  • More code to maintain than using AntD Table directly
  • Inline editing still DIY (but easier to compose)

Covers 100% of patterns but requires significant upfront investment to wire the rendering layer.

C

AG Grid Community/Enterprise

Most powerful. Full spreadsheet-like grid. Enterprise: $999/dev/year.

Pros

  • Every feature built-in: sort, filter, resize, reorder, edit, export
  • Best-in-class virtual scrolling (100k+ rows)
  • Column resizing and reordering work perfectly
  • Inline editing is first-class

Cons

  • Separate visual system — will never look like native AntD without constant theme maintenance
  • Row expansion requires Enterprise license ($999/dev/yr)
  • Excel export requires Enterprise license
  • Large bundle: 80-200KB
  • Subscription model — features stop if you stop paying

Not recommended. Visual mismatch with AntD is a dealbreaker for this project's goals.

D

Ant Design ProTable (@ant-design/pro-table)

Enhanced AntD Table. Auto-generated search forms, EditableProTable for inline editing.

Pros

  • Built on AntD Table — native theme integration
  • Auto-generates filter/search forms from column definitions
  • EditableProTable — best built-in inline editing of any option
  • valueType system reduces boilerplate (money, date, status, etc.)
  • MIT license

Cons

  • React 19 / Next.js 16 compatibility is uncertain — open issues
  • Bundles ALL ProComponents even if you only want ProTable
  • Documentation primarily Chinese, machine-translated English
  • Slower release cadence than core antd
  • Known pagination bugs with EditableProTable

Risky. React 19 compat and bundle bloat are concerns. Great if EditableProTable is essential.

My Recommendation

Option A (Ant Design Table) as the primary table component. It eliminates the copy-paste problem with a proper column definition system, handles 80% of your patterns natively, and has zero extra dependencies.

For the 20% edge cases: build a reusable <EditableTable> wrapper for inline editing, add SheetJS for export, and react-resizable for the rare table that needs column resizing.

If you later find AntD Table's limitations are blocking, TanStack Table (B) is the upgrade path — same column definitions, just swap the rendering engine.
