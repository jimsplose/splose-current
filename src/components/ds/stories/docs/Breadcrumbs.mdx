## When to use

Use Breadcrumbs on detail pages and nested settings routes where users need to understand where they are in the hierarchy and navigate back up more than one level — a patient note inside a patient record, an email template inside Settings > Templates, or a specific batch invoice inside the invoices list. Each item in the `items` array gets an `href` except the last, which is the current page (rendered with `aria-current="page"` and no link). When the path exceeds `maxItems`, middle entries collapse behind a `…` button that reveals them in a dropdown.

## When not to use

- **Navbar back arrow** — use instead for a single-level "go back" navigation where there is only one ancestor, and the full path does not need to be shown.
- **PageHeader** — use instead when you only need the current page title without a hierarchical path. Breadcrumbs and PageHeader can be composed: place Breadcrumbs above PageHeader for deep pages.

## Variants

- **Feature: Two-level** — simplest form: one ancestor link and one current-page label.
- **Feature: Three-level** — common in settings routes (Settings > Templates > Emails).
- **Feature: Four-level** — used in batch invoice flows (Invoices > Batch invoice > #330044 > Preview).
- **Feature: With Home Icon** — the root item accepts an `icon` node; use `<Icon as={HomeOutlined} size="xs" />` for a dashboard root.
- **Feature: Overflow (collapsed)** — set `maxItems` to collapse middle entries; `overflowDirection` controls whether the head, middle, or tail is hidden (default `"middle"`).
- **Feature: Alternative Separator** — pass `separator="/"` or any string/node to replace the default chevron icon.

## Composition

`Breadcrumbs` is used in `src/app/invoices/batch-invoice/[id]/page.tsx`:

```tsx
// src/app/invoices/batch-invoice/[id]/page.tsx
import { Breadcrumbs } from "@/components/ds";

<Breadcrumbs
  items={[
    { label: "Invoices", href: "/invoices" },
    { label: "Batch invoices", href: "/invoices" },
    { label: "BATCH-001" },
  ]}
/>
```

## Accessibility

The component renders a `<nav aria-label="Breadcrumb">` wrapping an `<ol>`. The current page item carries `aria-current="page"`. The overflow `…` button has `aria-label="Show hidden breadcrumbs"`. Item icons are marked `aria-hidden`. Consumers must ensure `href` values are valid — screen reader users rely on the link text to orient themselves.

## Sizing

Font size is fixed at 12 px for all items; the current page item is semi-bold (`fontWeight: 600`) while ancestor links are regular weight. The component wraps onto multiple lines when the container is narrow.

## Migration from

Replaces raw `antd Breadcrumb` imports with per-page separator overrides and manually duplicated `aria-current` handling scattered across detail page layouts.
