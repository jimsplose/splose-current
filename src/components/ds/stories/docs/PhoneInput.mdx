## When to use

Use PhoneInput for any phone number field in the application — patient mobile, clinic phone, emergency contact, location phone. The component wraps `react-phone-number-input` with an AntD input shell that matches `FormInput` visually, and always emits the value in E.164 format (e.g. `"+61412345678"`) regardless of how the user types it. This keeps stored values canonical across Australian, New Zealand, and international numbers without per-field normalisation code. It accepts controlled (`value` / `onChange`) and uncontrolled (`defaultValue`) modes, and threads `label`, `error`, `hint`, and `required` through `FormField`.

## When not to use

- **`<FormInput type="tel">`** — do not use for phone number fields; it does no country-aware formatting or E.164 normalisation, leading to inconsistent storage formats.

## Variants

- **Feature: Empty** — uncontrolled with no default value; shows the placeholder `0412 345 678`.
- **Feature: Filled (AU)** — pre-filled with an Australian number via `defaultValue`.
- **Feature: Disabled** — `disabled` prevents input and applies the disabled visual style from AntD.
- **Feature: Error** — `error` string renders beneath the field via `FormField` in the standard error style.
- **Feature: Default AU / NZ** — `defaultCountry` sets the initial country flag and dialling prefix (`"AU"` is the default; `"NZ"` for New Zealand clinics).
- **Feature: Sizes** — three tokens: `sm`, `md` (default), `lg`; forwarded to the inner AntD `Input`.

```tsx
import { PhoneInput } from "@/components/ds";

const [mobile, setMobile] = useState<string | null>(null);

<PhoneInput
  label="Mobile"
  value={mobile}
  onChange={setMobile}
  required
  hint="We'll use this for appointment reminders."
  defaultCountry="AU"
/>
```

## Accessibility

`PhoneInput` delegates label association, error announcement, and hint text to `FormField`, which renders a `<label>` linked to the input by `id`. The `required` prop adds an asterisk to the label. Error strings should be specific and actionable (e.g. "Enter a valid Australian mobile number.") rather than generic. Screen readers announce the `hint` as associated descriptive text. The country selector embedded by `react-phone-number-input` is not currently surfaced in this implementation — the country is set via `defaultCountry` at mount time only.

## Sizing

Three size tokens — `sm`, `md` (default), `lg` — scale the input height and font size to match the equivalent `FormInput` size. Use `md` for patient forms; `sm` for compact inline contexts.

## Migration from

Replaces `<FormInput type="tel">` usages with manual country handling and inconsistent E.164 storage, which produced different formats depending on how the user entered the number.
