# Splose Colour Token Reference for AntD Migration

**Date:** 2026-03-30
**Purpose:** Definitive colour reference for Claude Code to implement `theme.ts` and validate colour usage during the Ant Design migration.
**Source:** Figma colour audit artifact (Jim Yencken), cross-referenced with production SASS variables.

---

## How to Use This Document

When implementing colours in the AntD migration:

1. **Confirmed** tokens are the source of truth -- implement these.
2. **Updated** tokens had their hex value changed to match production -- use the hex shown here.
3. **Parked** tokens are NOT to be implemented yet -- they need a frontend audit or design decision first.
4. **Deprecated** tokens (all 1000-level shades) are reserved for potential dark mode and should NOT be included.
5. **New** tokens (cool-black family) were formalised from production variables and should be implemented.
6. When a token has a `Prod` source and a SASS variable mapping, that SASS variable is what production currently uses. The token name (e.g. `color-violet-600`) is the NEW canonical name.

**Resolved:** The original neutral-500/600 duplicate (#6E6E64) was analysed using OKLab perceptual lightness.
